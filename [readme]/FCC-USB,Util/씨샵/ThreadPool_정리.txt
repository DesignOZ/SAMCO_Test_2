TreadPool
: 사용자가 직접 스레드를 만드는게 아니고 기본적으로 CPU마다 제공되는 스레드를 활용하여 작업을 진행하는 방법. CPU마다 255개의 스레드가 제공됨.
즉 새롭게 스레드를 생성하는것보다 이 방법이 더 효율적으로 스레드자원을 이용할 수있다.


QueueUserWorkItem(WaitCallback)
: ThreadPool 클래스를 이용해서 메소드를 비동기적으로 실행하기 위해서 사용하는 메소드이다. 파라미터로는 delegate( 실제로 비동기적으로 실행하기위한 메소드를 참조함)을 갖게 된다.(오버로딩되어 2개의 변수를 받을 수 있긴하나 지금 알필요없으니 넘어감)

public delegate void WaitCallback(Object state)
: 이 delegate 는 반환타입이 없고 Object 타입으로 인자를 하나 받는다. 그래서 비동기적으로 실행하기 위한 메소드 또한 같은 반환타입과 인자여야만한다.


- QueueUserWorkItem 은 실행할 메소드를 ThreadPool의 큐에 쌓아 두고 실행 결과(큐에 잘 저장 되었는지 알려주는 결과)를 리턴한다. 

- 이렇게 쌓아둔 실행할 메소드들은 ThreadPool의 유효한 스레드가 생기면 바로 실행하는 식이다.

- 스레드를 할당 받을때까지 큐에 대기하고 있다가 실행되는 방식이다.
